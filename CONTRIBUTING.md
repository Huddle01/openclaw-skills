# Contributing to OpenClaw Skills

First off, thank you for considering contributing to OpenClaw Skills! It's people like you that make OpenClaw such a great and extensible platform.

## How Can I Contribute?

### 1. Reporting Bugs
If you find a bug in any of the existing skills or core functionality, please open an issue on GitHub.
*   **Use a clear and descriptive title** for the issue to identify the problem.
*   **Describe the exact steps** which reproduce the problem in as much detail as possible.
*   **Provide specific examples** to demonstrate the steps.

### 2. Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues. When you create an issue:
*   **Use a clear and descriptive title** for the issue to identify the suggestion.
*   **Provide a step-by-step description** of the suggested enhancement.
*   **Explain why this enhancement would be useful** to most OpenClaw Skills users.

### 3. Submitting a New Skill
We love seeing new tools and integrations! To add a new skill to this repository:
1. Fork the repository.
2. Create a new directory for your skill (e.g., `my-awesome-skill`).
3. Inside your skill directory, ensure you include:
   - The skill code, manifests, and configuration files.
   - A dedicated `README.md` explaining what the skill does and how to configure/install it.
4. Create a new branch for your feature (`git checkout -b feature/amazing-skill`).
5. Commit your changes (`git commit -m 'feat: add amazing skill'`).
6. Push to the branch (`git push origin feature/amazing-skill`).
7. Open a Pull Request.

## Code of Conduct
Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms. See [COMMUNITY.md](COMMUNITY.md) for more details.

## Pull Request Process
1. Ensure your code conforms to the existing style and quality guidelines.
2. Update the main `README.md` with details of your new skill to the list of available skills.
3. Your PR will require a review and at least one approval from a maintainer before it can be merged.
